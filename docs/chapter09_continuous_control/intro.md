# 第 9 章：连续控制——从LunarLander到MuJoCo

> **本章目标**：掌握连续动作空间中的 RL 方法。你将理解高斯策略与确定性策略的区别，以及 DDPG、TD3 等算法的设计动机。

## 10.1 动手：用 PPO 训练 HalfCheetah 奔跑

- 运行 MuJoCo 环境，看机器猎豹从原地打转到飞速奔跑
- 观察动作分布的演化：从方差很大的随机探索到精确的连续控制
- **你将获得**：连续动作空间 RL 的直观体验——输出不再是"左/右"，而是具体的力矩值

## 10.2 观察连续 vs 离散动作空间

- 连续动作空间：动作是实数向量（如各关节力矩 ∈ [-1, 1]）
- 离散动作空间：动作是有限集合（如 {左, 右}）
- 策略输出的区别：
  - 离散：输出每个动作的概率（Softmax）
  - 连续：输出动作的均值和方差（高斯分布），或者直接输出动作值（确定性策略）
- **你将思考**：连续空间中，你无法"穷举所有动作"，必须用参数化的策略

## 10.3 理论：连续动作的策略表示

- **高斯策略（Stochastic Policy）**：
  - 网络输出均值 μ(s) 和标准差 σ(s)
  - 动作从 N(μ, σ²) 中采样：a ~ π(·|s) = N(μ(s), σ(s)²)
  - 优点：天然带有探索（σ 控制探索程度）
  - 代表算法：PPO、A3C、SAC
- **确定性策略（Deterministic Policy）**：
  - 网络直接输出动作值：a = μ(s)
  - 优点：不需要处理策略分布，样本效率高
  - 缺点：没有内置探索机制，需要额外加噪声
  - 代表算法：DDPG、TD3
- **你将理解**：选择策略类型是连续控制的第一步决策

## 10.4 DDPG 与 TD3

- **DDPG（Deep Deterministic Policy Gradient）**：
  - 把 DQN 的思想搬到连续动作空间
  - Actor 网络输出具体动作值，Critic 网络评估 Q(s, a)
  - 问题：Q 值过估计 → 动作质量差 → 更差的 Q 值估计（恶性循环）
- **TD3（Twin Delayed DDPG）——三个关键改进**：
  1. **截断双 Q 学习（Clipped Double Q）**：训练两个 Critic，取较小值 → 解决过估计
  2. **延迟策略更新（Delayed Policy Update）**：Critic 更新多次后才更新 Actor → 让 Critic 先学准
  3. **目标策略平滑（Target Policy Smoothing）**：给目标动作加噪声 → 防止 Critic 在尖锐峰值处过拟合
- **你将理解**：TD3 的每个改进都在解决 DDPG 的一个具体问题——这是 RL 算法演进的典型模式

## 10.5 柔性演员-评论家：SAC (Soft Actor-Critic)

- **最大熵强化学习（Maximum Entropy RL）**：
  - 标准 RL 目标：最大化累积奖励 Σ γ^t r_t
  - SAC 目标：最大化 Σ γ^t (r_t + α H(π(·|s_t))) —— 既要得分高，又要动作分布尽可能随机（熵 H 大）
- **为什么 SAC 被视为现代连续控制的 Top 1 算法？**：
  - **优雅的探索-利用权衡**：如果多个动作得分差不多，策略就会保持随机（探索）；如果某个动作明显更好，策略才会变得确定（利用）。
  - **鲁棒性与样本效率**：作为 Off-policy 算法，它比 PPO 更省数据，比 DDPG/TD3 更稳定。
  - **温度参数 α 自动调节**：自动在探索和利用之间寻找平衡点。
- **你将理解**：SAC 是连续控制领域的“PPO”，工程界的默认首选。

## 10.6 回看实践：对比 PPO、TD3 与 SAC

- 在 HalfCheetah 上的对比实验：
  - **样本效率**：SAC / TD3 远超 PPO（因为它们是 off-policy，可以复用旧数据）
  - **探索能力**：SAC 的最大熵机制使其在复杂地形下表现更好
  - **训练稳定性**：PPO 依然是最不需要调参的（on-policy 的天然优势）
- 选择建议：大模型对齐选 PPO/GRPO，机器人连续控制选 SAC/TD3
- **你将获得**：主流连续控制算法的横向对比经验

## 10.7 并行采样加速

- 向量化环境（Vectorized Environment）原理：同时运行 N 个环境实例
- 实战：将训练速度提升 8 倍的配置技巧
  - num_envs = 8 → 每步获得 8 倍数据
  - 但注意：每个环境需要独立的随机种子，否则采样失去多样性
- 为什么 PPO 特别适合并行？——on-policy 算法需要大量新鲜数据，并行采样正好满足
- **你将获得**：RL 训练的工程加速技巧
