# 第 11 章：连续动作算法与具身智能

前面的章节里，我们的智能体一直在和"有限选项"打交道——CartPole 的"左推/右推"、Atari 的手柄按键、LLM 的 token 选择。这些场景有一个共同特征：动作是离散的、可数的，策略只需要回答"选哪一个"。

但真正的机器人控制不是这样的。一个六轴机械臂的每个关节需要在 $[-1, 1]$ 的连续范围内施加力矩，一辆自动驾驶汽车需要精确控制方向盘转角和油门开度，一只四足机器人的八个关节需要协调出流畅的步态。动作不再是"选 A 还是选 B"，而是"输出一个多维实数向量"。你不能穷举所有可能的力矩组合来挑最好的——因为实数有无穷多个。连续控制是 RL 走向物理世界的必经之路，而应对连续动作空间的算法思路，和我们之前学的 DQN、PPO 有本质区别。

本章从连续控制算法出发，最终走向具身智能——从仿真器中的策略训练，到真实机器人的部署。我们先在 PyBullet 仿真器里动手操控机器人，然后深入 SAC/TD3 等连续控制算法，最后讨论 Sim-to-Real 和具身智能的核心挑战。
