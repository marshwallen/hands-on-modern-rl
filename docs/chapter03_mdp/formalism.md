# 3.3 理论：MDP 五元组与价值函数

上一节我们用猜硬币游戏体验了 RL 的基本交互模式。现在，让我们把"状态、动作、奖励"这些概念提炼成一个精确的数学框架——马尔可夫决策过程（Markov Decision Process, MDP）。

这个框架由俄罗斯数学家安德烈·马尔可夫在 20 世纪初提出"马尔可夫链"的概念发展而来，后来在 1957 年被理查德·贝尔曼系统化为决策过程的数学框架 [^2]。不管你的智能体在下围棋、控制机器人、还是和人类对话，底层都可以用一个 MDP 来建模。

## MDP 五元组：(S, A, P, R, γ)

一个 MDP 由五个要素定义，通常写作 $(S, A, P, R, \gamma)$。

先看一张总览表，然后我们逐个拆解：

| 符号     | 名称     | 猜硬币           | CartPole                    |
| -------- | -------- | ---------------- | --------------------------- |
| $S$      | 状态集合 | {正, 反}         | $\mathbb{R}^4$（4维连续值） |
| $A$      | 动作集合 | {猜正, 猜反}     | {左推, 右推}                |
| $P$      | 转移概率 | 由硬币偏度决定   | 由物理引擎决定              |
| $R$      | 奖励函数 | 猜对 +1，猜错 -1 | 存活 +1                     |
| $\gamma$ | 折扣因子 | 本例中不涉及     | 通常 0.99                   |

**S（状态集合）**——"环境此刻长什么样"。状态是环境在某一时刻的完整描述。在猜硬币中，状态就是硬币当前是正还是反，只有 2 个可能的取值。在 CartPole 中，状态是一个 4 维向量 $[x, \dot{x}, \theta, \dot{\theta}]$，分别代表小车的位置、速度、杆子的角度和角速度。状态必须包含做决策所需的所有信息——这就是"马尔可夫"的含义。就像下象棋：只要你看清了当前的棋盘，你不需要知道之前是怎么走到这个局面的，就能做出最优决策。

**A（动作集合）**——"智能体能做什么"。猜硬币中只有"猜正"和"猜反"两个离散动作。CartPole 也是两个离散动作（左推/右推）。而在连续控制任务中（如机器人行走），动作可以是实数向量（如关节力矩）。

**P（转移概率）**——"世界如何运转"。$P(s' | s, a)$ 描述了：在状态 $s$ 下采取动作 $a$ 后，环境变成状态 $s'$ 的概率。在猜硬币中，不管你猜什么，硬币面都是独立重新抛出的。但在 CartPole 中，$P$ 由牛顿力学精确决定——施加一个力，小车就会按物理定律移动。这里有一个重要区分：在猜硬币中，$P$ 是已知的（因为我们自己写的代码）；但在大多数真实 RL 问题中，$P$ 是未知的——智能体不知道环境的精确运作方式，只能通过交互来"摸索"。

**R（奖励函数）**——"做得好不好"。$R(s, a)$ 告诉智能体这一步拿了多少分。奖励是 RL 中唯一的学习信号。CartPole 的奖励很简单（活着就 +1），但在大模型对齐中，"什么样的回答值得高分"本身就是一个研究课题——这正是 RLHF 和 DPO 要解决的核心问题。

**γ（折扣因子）**——"未来值多少钱"。$\gamma \in [0, 1]$ 控制智能体对"延迟满足"的重视程度：

$$G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots$$

当 $\gamma$ 接近 1 时，智能体非常"有远见"，愿意为了更大的未来回报而牺牲眼前利益。当 $\gamma$ 接近 0 时，智能体"目光短浅"，只在乎下一步能拿多少分。

为了加深直觉，让我们看看三种截然不同的 RL 任务的五元组：

| 要素     | 猜硬币       | CartPole               | LLM 对话               |
| -------- | ------------ | ---------------------- | ---------------------- |
| $S$      | {正, 反}     | $\mathbb{R}^4$（连续） | 已生成的 token 序列    |
| $A$      | {猜正, 猜反} | {左, 右}               | 词表中的下一个 token   |
| $P$      | 硬币偏度     | 物理引擎               | 语言模型自身的采样     |
| $R$      | ±1           | +1/步                  | 人类偏好/奖励模型打分  |
| $\gamma$ | —（单步）    | 0.99                   | 通常不使用（整句评估） |

尽管表面的差异巨大，但它们都是 MDP——都用同一套数学语言描述。这就是 MDP 框架的强大之处。

<details>
<summary>思考题：LLM 生成满足马尔可夫性吗？</summary>

严格来说，LLM 的自回归生成不满足传统意义上的马尔可夫性——下一个 token 的概率不仅取决于当前 token，还取决于整个上文。但我们可以把"已生成的完整序列"定义为状态 $s$，这样状态转移就只依赖于当前状态（完整上文）和动作（下一个 token），从而满足马尔可夫性。

在实际操作中，RLHF 通常把整个生成过程看作一个 episode，在序列结束后才给予奖励。这更接近蒙特卡洛方法，而非逐步 TD 学习。

</details>

## 3.4 价值函数 V(s) 与 Q(s, a)

有了 MDP 的框架，我们终于可以回答一个核心问题：这个局面到底值多少分？

要讲清楚价值函数，我们需要先理解一个基础概念——折扣累积回报 $G_t$，然后再一层一层地搭出 $V(s)$ 和 $Q(s, a)$。

### 第一块积木：折扣累积回报 $G_t$

在 RL 中，智能体关心的不是某一步的奖励，而是整个时间线上的总收益。但未来的奖励不能和眼前的奖励一视同仁——就像今天的 100 块钱比明年的 100 块更有价值一样，未来的奖励需要打一个折扣。

折扣累积回报 $G_t$ 的定义是：

$$G_t = r_t + \gamma \, r_{t+1} + \gamma^2 \, r_{t+2} + \gamma^3 \, r_{t+3} + \cdots$$

用一个具体例子来感受：假设 CartPole 每步奖励 +1，$\gamma = 0.9$，智能体活了 4 步后杆子倒了：

$$G_0 = \underbrace{1}_{r_0} + \underbrace{0.9 \times 1}_{r_1} + \underbrace{0.9^2 \times 1}_{r_2} + \underbrace{0.9^3 \times 1}_{r_3} = 1 + 0.9 + 0.81 + 0.729 = 3.439$$

同样是 +1 的奖励，但第 3 步的 1 分只值 $0.729$——因为 $\gamma = 0.9$ 连乘三次后，"未来的 1 分"贬值到不到原来的四分之三。$G_t$ 就是从现在开始到游戏结束，把所有奖励按"越远越不值钱"的规则累加起来的总分。

### 第二块积木：状态价值函数 $V^\pi(s)$

现在我们有了 $G_t$ 这个"总分"概念。但这里有一个问题：同样的状态、同样的策略，每次跑出来的 $G_t$ 可能不一样——因为环境的转移和策略本身都可能带有随机性。怎么办？取期望（平均值）。这就是状态价值函数 $V^\pi(s)$：

$$V^\pi(s) = \mathbb{E}_\pi \left[ G_t \mid s_t = s \right] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k \, r_{t+k} \;\middle|\; s_t = s \right]$$

$V^\pi(s)$ 的意思是：站在状态 $s$，遵循策略 $\pi$ 行动，平均能拿多少分。用一个类比来说，$V(s)$ 就像一个棋局评估器——给你一个棋盘局面，它告诉你从这步开始最终能赢的概率有多大。好的局面 $V$ 值高，差的局面 $V$ 值低。

在猜硬币中，如果硬币正面概率 60%、策略是"永远猜正"（$\pi(\text{猜正}) = 1$），那么：

$$V = 0.6 \times (+1) + 0.4 \times (-1) = 0.2$$

这个 0.2 就是：站在起点，遵循"永远猜正"的策略，平均每次能赢 0.2 分。

注意 $V$ 的上标是 $\pi$——这意味着同一个状态，用不同的策略会得到不同的价值。好策略的 $V$ 高，差策略的 $V$ 低。RL 的终极目标，就是找到让 $V$ 最高的那个策略 $\pi^*$。

### 第三块积木：动作价值函数 $Q^\pi(s, a)$

$V^\pi(s)$ 告诉你"这个局面整体值多少分"。但有时候你需要的不是整体的评分，而是"走这步棋之后值多少分"——这就是 $Q$ 函数：

$$Q^\pi(s, a) = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k \, r_{t+k} \;\middle|\; s_t = s, a_t = a \right]$$

和 $V$ 的唯一区别：$Q$ 多了一个条件 $a_t = a$——它固定了第一步的动作。继续用棋类来类比：

- $V(s)$：这个棋局整体胜率是多少？（不分走哪步）
- $Q(s, \text{出车})$：如果你走了出车这一步，之后的胜率是多少？
- $Q(s, \text{出马})$：如果你走了出马这一步，之后的胜率是多少？

显然，如果 $Q(s, \text{出车}) > Q(s, \text{出马})$，那你应该出车。比较不同动作的 $Q$ 值，就能做出更好的决策。

### V 和 Q 的关系

$V$ 和 $Q$ 之间有一个优雅的关系式。既然 $V(s)$ 是"不管走哪步整体值多少分"，而 $Q(s, a)$ 是"走某步之后值多少分"，那么 $V$ 自然就是所有 $Q$ 的加权平均——权重就是策略选择每个动作的概率：

$$V^\pi(s) = \sum_{a} \pi(a | s) \cdot Q^\pi(s, a)$$

用猜硬币来验证。假设硬币正面概率 60%，策略是"70% 猜正，30% 猜反"：

- $Q(s, \text{猜正}) = 0.6 \times 1 + 0.4 \times (-1) = 0.2$
- $Q(s, \text{猜反}) = 0.6 \times (-1) + 0.4 \times 1 = -0.2$
- $V(s) = 0.7 \times 0.2 + 0.3 \times (-0.2) = 0.14 - 0.06 = 0.08$

验算：每次 70% 概率猜正（期望 0.2）、30% 概率猜反（期望 -0.2），混合下来确实就是 0.08。状态的价值 = 所有动作价值的策略加权平均，就像一个班级的平均分等于各科成绩按学分加权平均一样。

### 为什么价值函数这么重要？

因为 RL 的核心任务就是估计 $V$ 或 $Q$。一旦你准确知道了每个状态的 $V$ 值（或每个状态-动作对的 $Q$ 值），最优策略就显而易见了——选 $Q$ 值最大的那个动作。这两个函数将在后续章节反复登场：第 5 章的 Critic 网络就是 $V(s)$ 的神经网络实现，第 6 章的 GAE 是对 $Q - V$（优势函数）的精巧估计，第 4 章的 DQN 直接学习 $Q(s, a)$ 的神经网络。

<details>
<summary>思考题：如果 V(s) 对所有状态都是 0，意味着什么？</summary>

如果 $V(s) = 0$ 对所有 $s$ 成立，说明当前策略在所有状态下都"既不赚也不亏"。这可能有两种情况：一是策略是纯随机的，环境也是公平的（如公平硬币的随机猜测）；二是策略太差了，以至于在好的局面和差的局面都拿不到正的回报。更一般地说，$V(s)$ 的相对大小比绝对值更重要——好状态的 $V$ 应该高于差状态的 $V$。

</details>
