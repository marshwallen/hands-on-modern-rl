# V(s) 与贝尔曼方程

上一节定义了 MDP 五元组 $\langle \mathcal{S}, \mathcal{A}, P, R, \gamma \rangle$、折扣累积回报 $G_t$ 和策略 $\pi$。有了这些工具，我们已经能精确地描述"游戏规则"和"优化目标"。但有一个关键问题还没有回答：**在当前局面下，智能体到底处于优势还是劣势？**

你可能会说："看奖励不就行了？"——但奖励只告诉你这一步拿了多少分。CartPole 中每步都是 +1，无法区分"杆子正在缓慢倾斜"和"杆子稳如泰山"。你需要的是一个更全局的评估——不仅看眼前的奖励，还要考虑"从这个局面出发，未来预期能拿多少总分"。这正是状态价值函数 $V(s)$ 的任务，它是所有 RL 算法的理论基石。

## V(s) 的定义

$V^\pi(s)$ 的正式定义是：在状态 $s$，遵循策略 $\pi$ 行动，所能获得的期望累积回报：

$$V^\pi(s) = \mathbb{E}_\pi \left[ G_t \mid s_t = s \right] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k \, r_{t+k} \;\middle|\; s_t = s \right] \tag{3.1}$$

上标 $\pi$ 表示价值依赖于策略——同一个状态，用不同策略会得到不同价值。好策略的 $V$ 高，差策略的 $V$ 低。RL 的终极目标，就是找到让 $V$ 最高的策略 $\pi^*$。

$V(s)$ 的直觉类似棋局评估器：给定一个棋盘局面，$V(s)$ 告诉你从这步开始最终能赢的概率有多大。好的局面 $V$ 值高，差的局面 $V$ 值低。AlphaGo 的核心组件之一就是一个 $V$ 网络——给它一个棋盘，它输出"黑方赢的概率"。这个数字不是随便猜的，而是通过数百万局自我对弈训练出来的。

以老虎机为例：只有一个状态，策略是"永远拉 A"（$\pi(\text{拉A}) = 1$），每步期望回报是 0.2，不考虑折扣时：

$$V = 0.6 \times (+1) + 0.4 \times (-1) = 0.2$$

但 $V^\pi(s)$ 的定义要求把从现在到永远的所有奖励加起来——你需要知道未来每一步会发生什么。对于 CartPole 的无限连续状态空间、大模型的天文数字 token 序列组合，穷举未来是不可能的。

## 贝尔曼方程的直觉：手画一张宝藏地图

贝尔曼方程（Bellman Equation）由美国数学家理查德·贝尔曼在 1957 年提出 [^1]，它提供了一个优雅的解决方案。在看公式之前，先通过一个练习建立直觉。

假设你在一个 1×5 的走廊里寻宝：

```
┌───┬───┬───┬───┬───┐
│ S │   │   │   │ 🏆 │
└───┴───┴───┴───┴───┘
```

规则：每走一步扣 1 分（消耗体力），到达宝藏 🏆 处不扣分（游戏结束），只能往右走。**从最右边开始，往左推，填出每个格子的价值。**

```
┌────┬────┬────┬────┬───┐
│ -4 │ -3 │ -2 │ -1 │ 0 │
└────┴────┴────┴────┴───┘
  S                      🏆
```

填写思路：🏆 那格价值 0；左边一格走一步就到，扣 1 分，价值 -1；再左边走两步，扣 2 分，价值 -2……依次类推。

现在考虑一个更有趣的场景：**如果走廊有岔路怎么办？**

```
          ┌───┬───┐
          │ -1│ 0 │ 🏆
      ┌───┼───┤   │
      │ -2│   │   │
┌────┬┤   │   │   │
│ S  ││   │   │   │
└────┴┴───┴───┴───┘
```

岔路意味着同一个格子可能有多个下一步选择。这时"下一格的价值"不再是唯一确定的——你需要考虑所有可能的目的地，按概率加权。这正是贝尔曼方程从"简单版"到"完整版"需要跨越的一步。记住这个直觉——我们马上就会看到它如何体现在公式中。

回到简单的直走廊，看第 2 格（值 -2）和第 3 格（值 -1）之间的关系：

$$\text{第 2 格的价值} = \underbrace{-1}_{\text{这一步的代价}} + \underbrace{\text{第 3 格的价值}}_{\text{走到那里之后还值多少}}$$

即 $-2 = -1 + (-1)$。每一格的价值，都等于"这一步的即时奖励"加上"下一格的价值"。没有看到未来所有的步骤——只看了一步，加上"下一格值多少"，就推出了整条走廊上所有格子的价值。这个思路就是贝尔曼方程。

## 贝尔曼期望方程

将宝藏地图的直觉推广到一般情况，贝尔曼期望方程的完整形式是：

$$V^\pi(s) = \sum_{a \in \mathcal{A}} \pi(a | s) \left[ R(s, a) + \gamma \sum_{s' \in \mathcal{S}} P(s' | s, a) \, V^\pi(s') \right] \tag{3.2}$$

逐项拆解：

| 符号 | 含义 |
| --- | --- |
| $V^\pi(s)$ | 状态 $s$ 在策略 $\pi$ 下的价值 |
| $\pi(a \| s)$ | 策略选择动作 $a$ 的概率 |
| $R(s, a)$ | 即时奖励 |
| $\gamma$ | 折扣因子 |
| $P(s' \| s, a)$ | 转移概率 |
| $V^\pi(s')$ | 下一状态的价值 |

一句话：**当前状态的价值 = 策略选各动作的概率 ×（即时奖励 + 折扣后的下一状态价值）。**

这个方程之所以重要，在于它把一个需要"看到无穷未来"的问题，转化成了一个递归结构：当前状态的价值只依赖于一步的即时奖励和下一状态的价值。

### 用老虎机验证

设定：A 台出奖率 60%，策略是"永远拉 A"，折扣因子 $\gamma = 0.9$。

老虎机是单状态的——不管拉多少轮，每一轮面对的情况完全相同，所以 $V$ 在每一轮都相同。代入贝尔曼方程：

$$V = \underbrace{[0.6 \times (+1) + 0.4 \times (-1)]}_{= 0.2} + \underbrace{\gamma}_{0.9} \times \underbrace{V}_{\text{同一状态}}$$

方程变成关于 $V$ 的一元方程：

$$V = 0.2 + 0.9V \implies 0.1V = 0.2 \implies \boxed{V = \frac{0.2}{1 - 0.9} = 2.0}$$

$\gamma$ 对 $V$ 的影响：$\gamma = 0$（只看眼前）时 $V = 0.2$；$\gamma = 0.9$（重视未来）时 $V = 2.0$；$\gamma \to 1$ 时 $V \to \infty$（无限期游戏累积奖励发散）。

## 贝尔曼最优方程

贝尔曼期望方程评估的是某个给定策略 $\pi$ 的价值。但 RL 的终极目标不是评估策略，而是**找到最优策略 $\pi^*$**。

贝尔曼最优方程只做了一个改动：把"对所有动作取策略加权平均"换成"选最好的那个动作"：

$$V^*(s) = \max_{a} \left[ R(s, a) + \gamma \sum_{s' \in \mathcal{S}} P(s' | s, a) \, V^*(s') \right] \tag{3.3}$$

两个方程的对比：

| | 贝尔曼期望方程 | 贝尔曼最优方程 |
| --- | --- | --- |
| **公式** | $\sum_a \pi(a\|s)[\cdot]$（加权平均） | $\max_a [\cdot]$（取最大值） |
| **含义** | "如果我按策略 $\pi$ 行动，能拿多少分" | "如果我每步都选最优，能拿多少分" |
| **依赖** | 需要知道 $\pi$ | 不需要 $\pi$ |

为什么最优方程里没有 $\pi$？因为 $\max_a$ 直接选价值最高的动作，不需要显式定义"以多大概率选哪个动作"。只要知道了最优价值 $V^*$ 或 $Q^*$，最优策略就确定了：$\pi^*(s) = \arg\max_a Q^*(s, a)$。

### 用一个具体例子验证贝尔曼最优方程

考虑一个有 3 个状态的简单 GridWorld：

```
┌─────┬─────┬─────┐
│  s1 │  s2 │  s3 │
│     │     │  🏆 │
└─────┴─────┴─────┘
```

规则：每走一步扣 1 分，到达 $s_3$（宝藏处）游戏结束，奖励为 0。只能往右走，$\gamma = 1$。

从右往左推：$V^*(s_3) = 0$（已经到了终点），$V^*(s_2) = \max_a [R(s_2, \text{右}) + \gamma \cdot V^*(s_3)] = -1 + 1 \times 0 = -1$，$V^*(s_1) = \max_a [-1 + V^*(s_2)] = -1 + (-1) = -2$。

现在假设在 $s_1$ 有两个选择：向右（走到 $s_2$，扣 1 分）和向上（撞墙，留在 $s_1$，扣 2 分）。贝尔曼最优方程告诉你：向右的价值是 $-1 + V^*(s_2) = -1 + (-1) = -2$，向上的价值是 $-2 + V^*(s_1) = -2 + (-2) = -4$。$\max_a$ 选向右——显然是更好的选择。这就是贝尔曼最优方程的核心逻辑：**在每个状态都做最优选择，得到的就是全局最优。**

<details>
<summary>思考题：$\max_a$ 看起来简单，实际有什么问题？</summary>

$\max_a$ 要求对所有动作都计算一遍再取最大。当动作空间很大（比如大模型有 5 万个 token 作为动作候选）时，这个"取最大"操作本身就变成了一个困难的计算问题。这正是路线一（Value-Based）在面对连续或超大动作空间时遇到的核心瓶颈——也是路线二（Policy-Based）存在的理由之一。

## V 的局限：不告诉你选哪个动作

$V(s)$ 告诉你一个局面值多少分，但它有一个根本性的局限：它不区分动作。$V(s) = 80$ 只说明"这个棋局不错"，但不告诉你"该走车还是走马"。要做出决策，你还需要更多信息——这正是 $Q(s, a)$ 要解决的问题。

但在讨论 $Q$ 之前，还有一个更紧迫的问题：贝尔曼方程给出了计算 $V$ 的理论公式，但在实际中，怎么从数据里把 $V$ 估计出来？你不知道环境的转移概率 $P$，也不可能穷举所有未来——那该怎么办？下一节将介绍三种经典方法——DP、MC、TD——每一代都解决了前一代的核心局限，以及贯穿全书的学习信号 TD Error。[经典方法速览：DP、MC 与 TD](./dp-mc-td)
