# 第 7 章：DPO——绕过奖励模型的魔法

> **本章目标**：理解 DPO 如何将 RLHF 问题转化为简单的分类损失，省去奖励模型训练这一工程瓶颈。你将掌握 DPO 的数学原理和实践技巧。
>
> **本章地位**：第 2 章是动手实战（"改个 Loss 就能训练"），本章则是"知其所以然"（The Math Behind the Magic）。你将看到 DPO 如何通过数学推导，巧妙地把包含 KL 散度惩罚的 RL 目标函数，**等价转换**成一个只需要人类偏好数据的交叉熵（分类）损失函数。看完你会感叹："难怪第 2 章只需要改个 Loss 就能绕过整个 PPO 循环！"

## 6.1 动手：用 DPO 对齐一个"毒舌"模型

- 准备一个小型偏好数据集（问题 + 好回答 + 坏回答）
- 运行 DPO 训练，每 100 步采样生成对比
- **你将获得**：DPO 的完整训练流程——比 RLHF 简单得多，不需要奖励模型

## 6.2 观察训练过程

- 模型从"阴阳怪气"逐渐变"礼貌得体"
- 训练 Loss 下降，但生成质量是否同步提升？——Loss 低 ≠ 回答好
- β（温度参数）的影响：太大则模型不敢改变，太小则训练不稳定
- **你将思考**：DPO 简单好用，但也有它的局限

## 6.3 理论：RLHF 的工程痛点（为什么需要 DPO？）

- 标准 RLHF 的三步流程：训练奖励模型 → 用 PPO 优化策略 → 部署
- 工程痛点：
  - 奖励模型训练耗时耗卡，需要大量标注数据
  - 奖励模型容易被"hack"——模型学会钻空子而非真正改善
  - PPO 训练需要同时维护 4 个模型（Actor、Critic、Reference、Reward）
- DPO 的核心思路：**跳过奖励模型，直接从偏好对中学**
- **回看第 6 章**：上一章我们花了大力气讲 Reward Model 怎么训练——Bradley-Terry 模型、成对标注、分布漂移。DPO 的核心问题是：**能不能不用 RM，直接从偏好数据中学？**
- **你将理解**：DPO 不是"更好的 RL"，而是"把 RL 问题重新表述为更简单的优化问题"
- **学术视角：DPO 本质上是一种 Offline RL**：
  - 回顾第 3 章的 on-policy vs off-policy 区分：PPO 是 on-policy（必须在线生成数据），而 DPO 连在线生成都不需要——它只用一个**固定的、预先收集好的偏好数据集**来训练
  - 在 RL 学术界，这叫 **Offline RL（离线强化学习）**：策略只能从固定数据集中学习，不能与环境交互
  - DPO 的巧妙之处在于：它不是把 offline RL 的难题（分布外动作、保守估计）硬扛下来，而是通过数学变换**完全绕过了 RL 的框架**，把问题变成了一个简单的分类问题
  - 这也是为什么 DPO 的工程复杂度远低于 PPO——没有在线采样循环，没有奖励模型，没有 Critic

### Online RL vs Offline RL：一图看懂

|                    | Online RL（PPO）           | Offline RL（DPO）              |
| ------------------ | -------------------------- | ------------------------------ |
| **数据来源**       | 当前模型实时生成           | 固定的离线偏好数据集           |
| **需要 RM 吗**     | 需要                       | 不需要                         |
| **需要在线交互吗** | 需要                       | 不需要                         |
| **类比**           | 边开车边学                 | 看录像学开车                   |
| **优势**           | 能动态探索，上限更高       | 工程简单，显存友好             |
| **代价**           | 4 个模型同时跑，极其吃资源 | 受数据质量限制，不能探索新策略 |

## 6.4 DPO 的数学推导——从 RL 目标到分类损失

- **起点**：RLHF 的原始目标是最大化带有 KL 惩罚的奖励
  - max E[r(x,y)] - β · KL[π_θ || π_ref]
  - 这个目标包含一个奖励函数 r（需要 RM）和策略 π（需要在线采样）
- **关键数学变换**（三步走）：
  1. **最优策略的解析解**：上述凸优化问题有闭式解——π\*(y|x) ∝ π_ref(y|x) · exp(r(x,y) / β)
  2. **反解奖励函数**：从最优策略表达式中反解出 r(x,y) = β log(π_θ(y|x) / π_ref(y|x))
  3. **代入 Bradley-Terry**：将反解出的 r 代入第 6 章的偏好模型，奖励函数完全消去，只剩策略概率的比值

- Bradley-Terry 偏好模型：人类偏好 y_w > y_l 的概率 = σ(r(x, y_w) - r(x, y_l))
- 关键数学变换：从奖励函数 r 到最优策略 π 的解析关系
  - π\*(y|x) ∝ π_ref(y|x) · exp(r(x,y) / β)
- 反解奖励函数：r(x,y) = β log(π_θ(y|x) / π_ref(y|x))
- 代入 Bradley-Terry 模型，得到纯策略概率表示的损失函数：
  - L = -log σ( β log(π*θ(y_w)/π_ref(y_w)) - β log(π*θ(y_l)/π_ref(y_l)) )
- 直觉解读：让好回答的概率相对参考模型上升，让坏回答的概率相对参考模型下降
- **回顾第 2 章**：看到这个公式了吗？这就是你在第 2 章代码里调的那个 `DPOTrainer` 背后的真正面目！当时你只需要改个 Loss 函数，背后是这一整套精巧的数学推导在支撑
- **你将理解**：DPO 的数学核心——用策略概率的比值**隐式地表示奖励**，把一个 RL 问题变成了一个简单的分类问题

## 6.5 隐式奖励

- DPO 训练后，模型内部实际蕴含了一个奖励模型：r(x,y) = β log(π_θ(y|x) / π_ref(y|x))
- 这意味着你可以用 DPO 训练后的模型来给任意回答打分
- 隐式奖励 vs 显式奖励模型：前者"免费"得到，后者需要额外训练
- **你将理解**：DPO 不是没有奖励模型，而是把奖励模型"藏"在了策略模型内部

## 6.6 回看实践：分析 DPO 训练指标

- Reward Accuracy：模型对偏好对的判别准确率（好回答得分 > 坏回答得分的比例）
- Margin：好回答与坏回答的 log 概率差（越大说明区分越明显）
- Chosen Reward 与 Rejected Reward 的走势：两者应该逐渐拉开距离
- **你将获得**：DPO 训练质量的诊断方法

## 6.7 DPO vs PPO：何时用哪个？

| 维度       | DPO                    | PPO                       |
| ---------- | ---------------------- | ------------------------- |
| 实现复杂度 | 低（纯分类损失）       | 高（4 个模型 + 在线采样） |
| 显存需求   | 低                     | 高                        |
| 数据需求   | 静态偏好对             | 在线采样 + 奖励模型       |
| 适用场景   | 固定数据集、小模型对齐 | 需要动态探索的复杂对齐    |
| 上限       | 受数据质量限制         | 理论上限更高              |

- **你将获得**：在项目中选择 DPO 还是 PPO 的决策框架

## 6.8 DPO 家族：KTO、SimPO 与 IPO

DPO 的成功催生了一整个"对齐方法族"——它们都基于同一个核心思想（绕过显式 RL），但在数据要求和数学细节上各有特色。

### KTO：只需要 👍/👎

**Kahneman-Tversky Optimization** 基于**前景理论**（行为经济学中的诺贝尔奖成果），大幅简化了数据收集：

|          | DPO                      | KTO                      |
| -------- | ------------------------ | ------------------------ |
| 数据格式 | 偏好对 (y_w, y_l)        | 单个标签 (y, 👍/👎)      |
| 收集成本 | 高（需要两个回答的比较） | 低（只需一个回答的评价） |
| 数学基础 | Bradley-Terry 偏好模型   | 前景理论的价值函数       |

KTO 的直觉：人类更容易给出"这个回答好不好"的二值判断，而不是"这两个回答哪个更好"的精细比较。

### SimPO：不需要参考模型

**Simple Preference Optimization** 进一步简化了 DPO：

- **去掉 reference model**：DPO 需要 $\pi_{ref}$ 作为基线，SimPO 用**回答自身的平均 log 概率**替代
- **省显存**：少维护一个完整模型，训练更快
- **理论依据**：隐式奖励 $r(x,y) = \beta \cdot \text{mean}(\log \pi_\theta(y|x))$，不需要参考 $\pi_{ref}$

### IPO：更稳健的优化

**Identity Preference Optimization** 用 KL 正则化替代 DPO 的 log-ratio：

- DPO 在偏好数据过拟合时会退化（把 good/bad 的概率差异推到极端）
- IPO 通过显式的正则化项防止这个问题
- 数学上更优雅，实践中在数据量少时更稳定

### 选型指南

| 场景             | 推荐                  | 理由                     |
| ---------------- | --------------------- | ------------------------ |
| 有大量偏好对数据 | DPO                   | 经典稳定，生态最好       |
| 只有👍/👎反馈    | KTO                   | 数据格式匹配，收集成本低 |
| 显存紧张         | SimPO                 | 不需要 reference model   |
| 数据量少         | IPO                   | 正则化防止过拟合         |
| 追求上限         | PPO / GRPO（第 8 章） | 在线方法上限更高         |
