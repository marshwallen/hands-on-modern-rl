# 第 10 章：LLM后训练工程全景

> **本章目标**：从工程视角完整走通 RLHF 的三阶段流水线。你将理解奖励函数设计、训练稳定性、奖励黑客防范等工业界每天都在面对的实际问题。
>
> **理论铺垫**：在动手之前，你需要知道 RLHF 三阶段背后的两个理论基础——**行为克隆（BC）**和**逆强化学习（IRL）**。SFT 本质就是 BC，RM 训练的思路来自 IRL。理解了这两个概念，RLHF 的每一步都不会再是黑箱。

## 11.0 理论基础：模仿学习——从专家示范中学习

- **核心思想**：RL 需要奖励信号来学习。但如果有一批"专家示范"（人类高手怎么做的），能不能直接从中学习，而不需要自己试错？
- **行为克隆（Behavioral Cloning, BC）**：
  - 方法：把专家的 (状态, 动作) 对当作监督学习数据，训练一个模型去模仿专家
  - 本质：就是**监督学习**——输入状态，输出专家会选的动作
  - **大模型中的对应物**：SFT（监督微调）本质上就是行为克隆！你给模型一堆"人类写的优质回答"，让它学会模仿人类的回答方式
  - BC 的致命问题：**分布漂移（Distribution Shift）**——模型犯一个小错 → 偏离专家的数据分布 → 后续看到的状态是训练时没见过的 → 犯更大的错 → 雪崩。这就是为什么 SFT 之后的模型还不够好，需要 RL 来"修复"BC 的累积误差
- **逆强化学习（Inverse Reinforcement Learning, IRL）**：
  - 问题翻转：不是"给定奖励，学习策略"（RL），而是"给定专家示范，反推奖励函数是什么"
  - 直觉：专家之所以这么做，说明他心中的奖励函数鼓励这种行为。IRL 的目标就是把这个隐含的奖励函数"挖"出来
  - **大模型中的对应物**：Reward Model 训练就是 IRL 的思想——从人类偏好的示范中，反推出"人类认为什么样的回答是好回答"的评分标准
  - IRL → RM 的演进：经典 IRL 需要反复迭代（学奖励 → 学策略 → 再更新奖励），RM 把它简化为一步（直接从偏好对中学一个评分函数）
- **从模仿学习到 RLHF 的完整逻辑链**：
  1. **BC = SFT**：从人类示范中学会基本的回答格式和能力（但有分布漂移问题）
  2. **IRL → RM**：从人类偏好中反推出"什么是好回答"的评分标准
  3. **RL = PPO/GRPO**：用 RM 当裁判，通过试错进一步优化策略（解决 BC 的分布漂移）
  - **你将理解**：RLHF 不是凭空发明的，而是"模仿学习的两步走"——先用 BC 跟着老师学基础，再用 RL 自己练习提升

## 11.1 训练数据工程：后训练的隐形支柱

工业界后训练工程师日常工作中，**70% 以上的时间花在数据上，而不是算法上**。这部分内容往往是论文里一笔带过、但实际工作中最耗精力的环节。

### SFT 指令数据构造

SFT 数据的质量直接决定模型"会不会按要求回答"。主流构造方法：

- **自指令法（Self-Instruct）**：给一个种子指令集，让模型自己生成新指令 + 回答，再人工过滤
- **进化式生成（Evol-Instruct）**：从简单指令出发，通过"加深""加宽""具体化"等操作逐步生成更复杂的指令
- **难度分级**：将数据按难度分成多个桶（easy/medium/hard），不同阶段用不同难度的数据
- **数据配比策略**：不同任务数据（对话/代码/数学/创作）的比例会显著影响最终效果。常见做法是先做小规模消融实验确定配比，再大规模训练

### 偏好数据生成

偏好数据（chosen/rejected 对）是 DPO 和 RLHF 的核心燃料：

- **人工标注**：标注员从模型的多个输出中选择最好的。成本约 $0.5-5/对，质量高但速度慢
- **AI 交叉打分**：用多个模型对同一组回答独立打分，取排名差异构造偏好对
- **LLM-as-Judge**：用强模型（如 GPT-4）对回答评分，自动生成偏好对
- **在线偏好采集**：从实际用户交互中收集隐式偏好信号（点赞/踩/编辑后重发）

### 数据清洗与质量过滤

原始数据总是脏的，清洗流程包括：

- **去重**：语义去重（ embedding 相似度 > 阈值的视为重复）
- **去噪**：过滤掉低质量回答（过短、重复、无意义）
- **安全过滤**：移除有害、涉密、版权内容
- **难度过滤**：确保数据难度分布合理——全部太简单模型学不到东西，全部太难模型学不动

### 数据合成（Synthetic Data）

2026 年最热门的方向之一——用 AI 批量生成训练数据：

- **思维链合成**：用强模型生成带推理步骤的回答，蒸馏到小模型
- **工具调用轨迹合成**：自动生成"用户提问 → 模型调用工具 → 获取结果 → 综合回答"的完整轨迹
- **多步推理数据**：通过搜索 + 验证生成高质量的数学/代码推理链
- **代表性工具**：Magpie（自动指令生成）、Self-Instruct、Evol-Instruct

### 数据闭环

工业界的实际工作流不是"训练一次就完事"，而是一个持续迭代的数据闭环：

```
确定目标能力
    ↓
构造/合成训练数据
    ↓
模型训练（SFT / RLHF / GRPO）
    ↓
评测（Benchmark + Badcase 分析）
    ↓
定位薄弱环节 → 补充针对性数据
    ↓
重新训练 → 重复迭代
```

每一次迭代都是"数据 → 训练 → 评测 → 数据"的闭环。**数据闭环的运转效率，往往比单次训练的算法选择更重要。**

## 11.2 动手：从头跑通 RLHF 三阶段（玩具规模）

- **阶段一 SFT**：用指令数据微调基础模型，让它学会"按要求回答"
- **阶段二 RM**：训练一个奖励模型，输入 (问题, 回答)，输出一个标量分数
- **阶段三 PPO**：用奖励模型给生成回答打分，用 PPO 优化语言模型
- 对比三个阶段模型的输出变化
- **你将获得**：RLHF 完整 pipeline 的端到端体验

## 11.2 理论：奖励函数设计专题

- **规则奖励（Rule-based Reward）**：
  - 优点：确定性强、不会被 hack、成本为零
  - 缺点：只能检查格式/语法，无法评估语义质量
  - 典型用法：答案正确性检查、格式规范度评分、代码是否能运行
- **模型奖励（Model-based Reward）**：
  - 优点：能理解语义、覆盖面广
  - 缺点：训练成本高、可能被 hack、存在偏见
  - 典型用法：人类偏好对齐、回答质量评估
- **奖励黑客（Reward Hacking）案例**：
  - 长度偏见：模型学会写长篇大论来获取高分 → 加入长度惩罚
  - 重复废话：模型反复重复安全但无意义的话 → 加入重复惩罚
  - 格式作弊：模型用特殊 token 骗取格式奖励 → 加强验证逻辑
- **多维度奖励与动态权重**：R_total = w_1·R_helpfulness + w_2·R_safety + w_3·R_format
- **你将理解**：奖励函数 = RL 系统的"目标函数"，设计不当会导致南辕北辙

## 11.4 奖励模型深入：从训练到部署

### 奖励模型的训练方法

奖励模型本质上是一个回归模型——输入 (prompt, response)，输出一个标量分数。但训练过程有不少工程细节：

**数据标注**：

- 标注员对同一个 prompt 的多个回答进行排序（不是打绝对分，而是排序——人类更擅长比较）
- 通常每个 prompt 收集 4-9 个回答的排序
- 排序成本远低于写参考答案，但仍需要专业培训的标注团队

**损失函数设计**：

- 经典方法使用 **Bradley-Terry 模型**：$L = -\log \sigma(r_\theta(x, y_w) - r_\theta(x, y_l))$
- 其中 $y_w$ 是标注员偏好的回答，$y_l$ 是不偏好的回答
- 多回答排序时推广为：$L = -\sum_{i<j} \log \sigma(r_\theta(x, y_i) - r_\theta(x, y_j))$，利用所有排序对

**训练技巧**：

- 基座模型通常选择与策略模型相同架构但较小规模的模型（如 7B 策略用 3B 做 RM）
- 在 SFT 模型基础上微调，而非从 base 模型开始
- 学习率通常比 SFT 更保守（1e-5 到 5e-6）

### 细粒度奖励：Token-level vs Sequence-level

| 粒度            | 方式                                 | 优劣                                 |
| --------------- | ------------------------------------ | ------------------------------------ |
| **Sequence**    | 整个回答一个 reward 分数             | 简单，但无法区分好回答中哪些部分好   |
| **Token-level** | 每个 token 都有独立 reward           | 精细，但训练成本高且信号噪声大       |
| **Step-level**  | 按推理步骤分（一段 CoT 一个 reward） | 折中方案，也是 PRM（过程监督）的思路 |

实践中最常用的是 **sequence-level**（PPO/GRPO 默认）+ **规则辅助**（格式、长度等 token-level 规则奖励）。

### 规则奖励 vs 模型奖励的选择

|              | 规则奖励                             | 模型奖励（RM）               |
| ------------ | ------------------------------------ | ---------------------------- |
| **成本**     | 几乎为零                             | 训练 + 推理成本              |
| **可靠性**   | 确定、不可被 hack                    | 可被 hack（reward hacking）  |
| **语义理解** | 无（只能检查格式/答案正确性）        | 有（能理解内容质量）         |
| **适用场景** | 数学/代码等有客观答案的任务（RLVR）  | 对话/创意/安全等主观偏好任务 |
| **最佳实践** | 与模型奖励组合使用，形成混合奖励信号 |                              |

典型的混合奖励：$R_{total} = R_{RM} + \alpha \cdot R_{format} + \beta \cdot R_{length} + \gamma \cdot R_{correctness}$

## 11.5 训练稳定性技巧

- **KL 散度惩罚**：在 PPO 目标中加入 β·KL(π_θ || π_ref)
  - 作用：防止模型偏离参考模型太远（不会"忘了怎么说人话"）
  - 调参：β 太大 → 训练不动；β 太小 → 模型可能退化
- **混合训练**：PPO Loss + α × SFT Loss
  - 作用：在 RL 训练的同时保持监督学习的信号，防止能力遗忘
- **梯度裁剪与学习率调度**：
  - 梯度裁剪：防止梯度爆炸导致训练崩溃
  - 余弦退火：让学习率逐步下降，训练后期更精细
- **你将获得**：让 RLHF 训练"不崩"的工程技巧清单

## 11.6 回看实践：分析 RLHF 各阶段指标

- **SFT 后**：格式对了（知道怎么回答问题），但内容泛泛（没有偏好判断）
- **RM 后**：能打分（知道哪个回答更好），但自己不会写好回答（判官 vs 考生的区别）
- **PPO 后**：会写"讨好 RM"的回答（RM 给高分的回答风格被强化）
- 关键发现：PPO 后模型的回答可能比 SFT 后更"谄媚"——这是 RLHF 的已知副作用
- **你将获得**：理解 RLHF 每个阶段的输入输出，以及它们之间的信息流

## 11.7 如何判断奖励黑客？

- **信号一**：回答长度与奖励呈强正相关 → 模型可能在"凑字数"
- **信号二**：人工评估分数 vs 模型奖励分数的散点图偏离对角线 → 奖励模型与人类判断不一致
- **信号三**：特定短语高频出现 → 模型可能发现了某个"高分咒语"
- **信号四**：训练集上 reward 持续上升但验证集上质量下降 → 过拟合奖励函数
- 修复方法：多维度奖励、对抗性数据标注、定期人工抽检
- **你将获得**：诊断和防范奖励黑客的实战检查清单

## 11.8 RLAIF：用 AI 替代人类标注

RLHF 的最大瓶颈是**人类标注成本**。RLAIF（Reinforcement Learning from AI Feedback）用 AI 替代人类来做偏好判断，大幅降低成本。

### Constitutional AI：AI 自己定规矩

Anthropic 提出的 Constitutional AI（CAI）是 RLAIF 的代表性方法：

1. **自我批评**：给模型一个回答，让它按照"宪法原则"（如"不要有害""要诚实"）进行自我批评
2. **自我修订**：基于批评结果，让模型修改自己的回答
3. **AI 偏好生成**：用修改前后的回答构造偏好对
4. **训练**：用这些 AI 生成的偏好对训练模型（可以用 DPO 或 RL）

整个过程**不需要任何人类标注**。

### Self-Play：模型自我博弈

更激进的方案是让模型通过自我博弈来提升：

- **SeRL**（NeurIPS 2025）：模型自己生成问题和回答，自己评估质量，自己更新策略
- **SAO**（Self-Alignment Optimization）：完全自合成的训练数据管线，所有数据都是模型生成的
- **Self-Play Debate**：两个模型互相辩论，裁判模型评判胜负，赢家被强化

### RLAIF vs RLHF 的对比

|          | RLHF              | RLAIF                           |
| -------- | ----------------- | ------------------------------- |
| 反馈来源 | 人类标注员        | AI 模型                         |
| 成本     | 极高（$0.5-5/对） | 极低（推理成本）                |
| 可扩展性 | 受标注速度限制    | 几乎无限                        |
| 质量风险 | 人类偏见          | AI 偏见（可能放大模型已有缺陷） |
| 适用场景 | 高质量对齐        | 大规模快速迭代                  |

### 实践建议

1. **先用 RLHF 建立高质量的"种子"数据**
2. **再用 RLAIF 做大规模扩展**
3. **定期用人类评估校准 AI 的判断质量**
4. **警惕"AI 反馈导致的模型退化循环"**：如果 AI judge 本身有偏见，会不断强化这个偏见
