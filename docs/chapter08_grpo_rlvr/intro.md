# 第 8 章：GRPO、DAPO 与 RLVR——从策略优化到可验证奖励

> **本章目标**：理解 GRPO 如何通过"组内相对比较"省掉 Critic 网络，DAPO 如何进一步优化采样效率，以及 RLVR 如何用可验证奖励取代人工标注。这三者共同构成了 2025-2026 年 LLM RL 训练的核心技术栈。
>
> **本章暗线**：第 5 章我们学会了 Actor-Critic 架构——用 Critic 网络提供基线来降低方差。第 6 章的 PPO 把它用到了极致——同时跑 Actor、Critic、Reference、Reward 四个模型。现在 DeepSeek 说："Critic 太占显存了，我不要了！"GRPO 的核心创新就是：**用组内的均值和标准差自己算 Baseline，干掉 Critic 网络。** 而在奖励端，RLVR 则说："连 Reward Model 都不需要！"——直接用客观可验证的信号做 reward。整个逻辑链条完美闭环。

## 7.1 动手：用 GRPO 训练数学推理

- 环境：GSM8K 数学题 + 规则奖励（答案正确 +1，格式规范 +0.5）
- 运行开源 GRPO 实现，观察训练曲线
- **你将获得**：GRPO 训练的完整体验——与 PPO 相比，少了一个 Critic 模型

## 7.2 观察训练过程

- 显存占用比 PPO 少多少？——省掉 Critic 通常减少 30-40% 显存
- 同一问题，训练前后推理步骤有何变化？——从"直接猜答案"到"先列算式再计算"
- 组内方差的变化：训练初期回答质量差异大，后期组内趋于一致
- **你将思考**：组内归一化为什么能替代 Critic 的作用？

## 7.3 理论：GRPO 的动机

- PPO 的 Critic 在 LLM 训练中的三大问题：
  1. **吃显存**：Critic 本身就是一个与 Actor 同等规模的模型
  2. **训练不稳定**：价值函数在长序列上的估计方差很大
  3. **工程复杂**：需要额外维护 Critic 的优化器、学习率调度等
- GRPO 的核心思路：**用组内相对比较替代绝对价值估计**
- 类比：不问"这个回答得几分"，而是问"这几个回答里哪个最好"
- **你将理解**：GRPO 的创新不在于网络架构，而在于"如何估计好坏"这个根本问题

## 7.4 GRPO 核心机制

- 对每个问题 x 采样 k 个回答 {y_1, y_2, ..., y_k}（Group）
- 用规则或模型对每个回答打分 {r_1, r_2, ..., r_k}
- 计算组内归一化优势：A_i = (r_i - mean(r)) / std(r)
  - 比平均好的回答 A > 0 → 增加概率
  - 比平均差的回答 A < 0 → 降低概率
- 使用与 PPO 相同的裁剪目标更新策略
- **你将理解**：GRPO = "PPO 的裁剪机制 + 用组内排名替代 Critic"

## 7.5 为什么组内归一化有效？

- **题目难度被自动归一化**：简单题的回答普遍得高分，减去均值后只剩下"这道题内部谁更好"
- **回答之间的相对优劣才是关键**：人类偏好本质上也是比较式的（A 比 B 好），不是绝对的（A 得 87 分）
- **方差更低**：组内比较比跨样本的绝对评分更稳定
- 局限性：需要多个回答（k 通常取 4-16），采样成本更高
- **你将理解**：相对比较 vs 绝对评分——这是理解 GRPO 的关键视角

## 7.6 回看实践：分析 GRPO 训练曲线

- Reward 均值上升 vs 组内方差变化
- 回答长度是否会失控？（需要长度惩罚项）
- 不同 k 值（组大小）的影响：k=4 vs k=8 vs k=16
- **你将获得**：GRPO 超参数调优的实践经验

## 7.7 DeepSeek-R1-Zero 范式：大模型纯 RL 强化

- **行业铁律被打破**：传统 RLHF 认为”Base 模型必须先做 SFT（学会说人话）才能做 RL，否则探索空间太大模型会崩溃”。
- **Zero 范式的颠覆**：DeepSeek-R1-Zero 证明了，在拥有明确客观规则奖励（如代码编译通过、数学答案匹配）的领域，**完全不需要 SFT 进行冷启动**。直接让 Base 模型进行大规模 GRPO 训练是可行的！[📖 _DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning (arXiv:2501.12948)_](https://arxiv.org/abs/2501.12948)
- **涌现（Emergence）与顿悟（Aha Moment）**：
  - 在纯 RL 训练过程中，模型自主发展出了长思维链（CoT）。
  - 模型甚至学会了在回答错误时自我反思、推翻前言并重新计算。这并非人类教给它的，而是模型为了拿到更高的 Rule-based Reward 自己”领悟”出的高分策略。
- **DeepSeek-R1 的多阶段工程妥协**：
  - 虽然 R1-Zero 证明了纯 RL 的可行性，但它的回答往往语言混杂、格式较差。
  - 因此最终版 DeepSeek-R1 采用了妥协的多阶段流程：冷启动少量高质量 SFT → 大规模 GRPO 强化推理 → 拒绝采样筛选数据 → SFT 强化格式与人类偏好 → 再次结合 RM 进行 PPO/GRPO 训练。

## 7.8 DAPO：GRPO 的下一代演进

GRPO 已经证明了”不需要 Critic 也能做 RL”，但它仍有几个工程痛点。DAPO（Decoupled Clip and Dynamic Sampling Policy Optimization）针对性地解决了这些问题。

### DAPO 的四大改进

| 改进                        | GRPO 的问题                                 | DAPO 的解法                               |
| --------------------------- | ------------------------------------------- | ----------------------------------------- |
| **Clip-Higher**             | 上下对称裁剪，低概率动作被过度抑制          | 解耦裁剪范围，给低概率动作更多上升空间    |
| **动态采样**                | 所有 prompt 都参与训练，浪费算力            | 过滤掉模型已经答对的 prompt（零梯度样本） |
| **Token 级损失**            | 序列级 reward 归一化，忽略了 token 间的差异 | Token 级策略梯度，更精细的信用分配        |
| **Overlong Reward Shaping** | 过长回答被直接截断惩罚，梯度信号不连续      | 平滑的长度惩罚，避免训练不稳定            |

### DAPO 的效果

DAPO 在 AIME 2024 数学竞赛上用 DeepSeek-R1 **一半的训练步数**就达到了 50 分。它被 NeurIPS 2025 接收为 poster，标志着 GRPO → DAPO 的明确演进路线。

> **算法演进全景**：PPO（需要 Critic + RM）→ GRPO（不需要 Critic）→ DAPO（不需要 Critic + 动态采样 + 更精细的信用分配）

## 7.9 RLVR：可验证奖励强化学习

### 核心思想

RLVR（Reinforcement Learning with Verifiable Rewards）是 2025 年最重要的 RL 范式转变：

```
传统 RLHF：  Prompt → 模型回答 → Reward Model 打分 → 更新
RLVR：       Prompt → 模型回答 → 执行验证器 → 对/错（0/1） → 更新
```

**不需要人类标注，不需要训练 Reward Model**——直接用客观可验证的信号做 reward。

### 为什么 RLVR 如此重要？

| 方面        | RLHF                   | RLVR                         |
| ----------- | ---------------------- | ---------------------------- |
| 数据成本    | 极高（人类标注偏好对） | 极低（自动验证）             |
| Reward 质量 | 有噪声（人类主观性）   | 精确（客观对错）             |
| 可扩展性    | 受标注速度限制         | 几乎无限                     |
| 适用范围    | 主观偏好（礼貌、安全） | 客观任务（数学、代码、逻辑） |

### RLVR 的验证器设计

| 领域       | 验证方式   | 示例                        |
| ---------- | ---------- | --------------------------- |
| 数学       | 答案匹配   | `\\boxed{42}` == 标准答案   |
| 代码       | 单元测试   | 代码执行 + test case 通过率 |
| 逻辑推理   | 形式化验证 | Lean/Coq 定理证明器         |
| 多语言翻译 | BLEU/COMET | 自动评分指标                |

### RLVR 的局限与争议

1. **只适用于有客观答案的领域**：RLVR 无法处理”更礼貌””更有创意”这类主观偏好
2. **验证器可能被 hack**：模型可能学会满足验证器但不真正理解
3. **”RLVR 真的提升推理能力吗？”**：2025 年 NeurIPS 的一篇 oral 论文质疑，RLVR 可能只是在提高搜索效率（test-time scaling），而非注入真正的推理能力——这是一个开放的前沿争议

### 1-Shot RLVR：惊人的发现

更令人惊讶的是，ICLR 2025 的研究表明 RLVR **只用 1 个训练样本**就能工作。这说明 RLVR 的有效性不依赖于数据量，而是 RL 框架本身”解锁”了模型已有的推理潜力。

## 7.10 Online vs Offline：LLM 对齐的实践选择

在学完 DPO（Offline）和 GRPO/DAPO（Online）之后，一个自然的实践问题是：**该选哪个？**

### 三种训练范式对比

|            | Offline (DPO)      | Online (PPO/GRPO)        | Semi-Online           |
| ---------- | ------------------ | ------------------------ | --------------------- |
| 数据       | 固定偏好数据集     | 实时采样生成             | 离线数据 + 周期性更新 |
| 探索       | 无（被数据集限制） | 有（模型自己探索新策略） | 部分                  |
| 上限       | 受数据质量限制     | 理论上更高               | 折中                  |
| 工程复杂度 | 低                 | 高                       | 中                    |
| 代表方法   | DPO, KTO           | PPO, GRPO, DAPO          | Iterative DPO, RLOO   |

### 实践建议

1. **快速原型**：先用 DPO 验证数据质量是否足够好
2. **提升上限**：在 DPO 基础上切换到 GRPO/DAPO 做在线优化
3. **工程折中**：如果资源有限，用 Semi-Online（Iterative DPO）是性价比最高的选择

### 你将理解

PPO → GRPO → DAPO 是策略优化端的演进，RLHF → RLVR 是奖励端的演进，Online → Offline 是工程端的权衡。这三个维度可以独立选择、灵活组合。

## 7.11 RL Scaling：用更多算力换更强推理

2025 年最令人兴奋的发现之一：**RL 训练的收益尚未饱和**。继续加大 RL 训练规模（更多数据、更多采样、更长训练），模型的推理能力还在持续提升。

### RL Scaling 的三个维度

| 维度         | 含义                           | 实践方法                                 |
| ------------ | ------------------------------ | ---------------------------------------- |
| **数据规模** | 更多不同难度的训练 prompt      | 自动生成 + 筛选高质量数学/代码题         |
| **采样规模** | 每个 prompt 采样更多回答（k↑） | k 从 4 增加到 16 甚至 64，组内比较更稳定 |
| **训练步数** | 更长时间的 RL 训练             | 监控 KL 散度和评测指标，防止过度训练     |

### Scaling 的关键发现

- DeepSeek-R1 证明：在数学推理上，RL 训练的 scaling 曲线比 SFT 更陡峭
- 继续增加训练步数，模型的 pass@1 持续提升，尚未观察到明确饱和
- **关键前提**：需要足够多样化的 prompt 数据，否则模型会过拟合到训练集

## 7.12 Test-time Scaling：推理时消耗更多算力

与 RL Scaling（训练时投入更多算力）互补的另一条路线：**推理时也让模型"多想一想"**。

### 核心思路

```
标准推理：   Prompt → 模型直接输出答案
Test-time：  Prompt → 生成多个候选 → 验证/投票 → 选最佳
```

### 主要方法

| 方法                          | 原理                             | 适用场景              |
| ----------------------------- | -------------------------------- | --------------------- |
| **Best-of-N 采样**            | 生成 N 个回答，选 reward 最高的  | 简单直接，成本线性增  |
| **多数投票（Majority Vote）** | 生成 N 个回答，选出现最多的答案  | 数学/代码等有确定答案 |
| **MCTS / Tree of Thought**    | 在推理空间做树搜索，回溯错误分支 | 复杂推理任务          |
| **Verifier-guided**           | 用验证器在推理过程中动态剪枝     | 代码/数学             |

### RL 与 Test-time Scaling 的关系

一个开放的前沿争议是：**RLVR 是否只是提升了 test-time 的搜索效率，而非注入了真正的推理能力？**

- 支持方：RL 训练后的模型在相同采样预算下表现更好，说明 RL 确实改变了模型的内部策略
- 质疑方：未经过 RL 训练的基础模型，如果给予足够多的采样次数（N→∞），理论上也能达到类似效果
- 现实：实践中 RL 训练带来的效率提升是巨大的——让模型在少量采样时就能输出高质量回答

## 7.13 PRM vs ORM：过程监督与结果监督

这是第 12 章（Agentic RL）中信用分配问题在推理场景的具体化。

### ORM（Outcome Reward Model）

只看最终结果对不对：

- 优点：标注简单（只需要知道最终答案对错），成本低
- 缺点：信号稀疏——7 步推理中只有最后一步有反馈，中间步骤的对错无从得知

### PRM（Process Reward Model）

每一步推理都评估：

- 优点：学习信号密集，能精确指导每一步的改进方向
- 缺点：标注成本极高（需要人类专家逐步判断每步推理是否正确）
- 代表性工作：OpenAI 的 PRM800K 数据集（数学推理每步标注）

### 自动化 PRM 的探索

由于人工逐步标注太贵，研究者开始探索自动化的过程监督：

- **自动 PRM**：用蒙特卡洛采样估计每一步的正确概率——从第 i 步开始重新采样 N 次后续推理，看最终答案正确的比例
- **ORM → PRM 蒸馏**：先训练 ORM，再用 ORM 对每一步打分作为 PRM 的训练数据
- **RL 自监督 PRM**：让模型在 RL 训练过程中同时学习评估自己的推理步骤

### PRM 的实际效果

| 方法     | GSM8K 准确率 | MATH 准确率 | 标注成本 |
| -------- | ------------ | ----------- | -------- |
| 仅 ORM   | ~82%         | ~40%        | 低       |
| 仅 PRM   | ~85%         | ~45%        | 极高     |
| ORM + RL | ~88%         | ~50%        | 低       |
| PRM + RL | ~90%         | ~55%        | 极高     |

PRM 的提升是实打实的，但成本也是实打实的。实践中通常先用 ORM 快速验证 RL 训练流程，再考虑引入 PRM 做精细优化。
